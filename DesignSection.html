Goal Alignment

This Website was designed with a future employer as the user in mind. I wanted the Max Kruger Website to be easily traversable and display everything that a future employer may need or want to know about me. Basically, I wanted to make an employers work as easy as possible. As an employer they would want to know exactly who I am, what type of person I am, my goals, my achivements, and my interests.

In my portfolio section, you can find out most things about me, my interest in Game Development, other hobbies, my strengths, weaknesses. Anything concerning me as a person is found there and is the most important for an employer.
This website itself portrays my skills in front-end development. But if you want to know more about the creation of this website and my learning experience in HTML and CSS then check out the Blog Section!

User Flow

Explanation of Interaction Points

The main interface element in my website is an a tag that is used to traverse between different pages. These tags look like buttons with a rounded border. These tags are extremely readible and contrast the background colours. I also have a search bar for if users are searching for something specific and they do not wish to navigate through menus. These are both navigational components. Lastly I am using a drop down menu as a way for employers to contact me personally or to find my social network accounts. This is a Input Control.

So I decided to change my paragprahs font to Times New Roman It has good readibility and legibility throughout the website. I use predominantly white and purple, especially in headings and buttons. But smaller h3 headings have purple backgrounds and white font colours. This is consistent throughout my website. With regards to hierarchy I use size, colour and whitespace to represent important headings and buttons. I use proximity and repetition throughout my website so there is a constant sense of distance and repetition between all of my elements. Lastly, my interface elements use a rounded shape to create a friendly voice and tone to them, inviting the user to read more and navigate easily. But this is balanced by my selection of purple which cteates a more serious tone so that employers may know that although this is an inviting website, I am serious about my work and presenting myself. All headings are placed above paragraph text to pull the eye toward it first. The white background of headings contrast the purple background of my website, highlighting its importance in the hierarchy and again, pulling the users eye toward it first.

Implementation notes

a) My folder structure is laid out to sepearate the html and CSS documents and they are all in order of the hierarchy of the actual site. I used appropriate naming conventions for all styling and made sure to not use in-line styling anywhere throughout my site. The coding is modular in that it can be easily upgraded or changed completely.

b) I have tried my best in using semantic markup and have managed to clearly express my content through different containers, buttons, links and pages. Using Semantic markup makes my webiste optimised for an search engine. If someone were to search "Max Kruger Game Development", a search engline like google would be able to display my site as one of the answers. However, I could definately improve the accessibility of my website by creating more semantic tags to guide, for example, a blind person using an Ereader to navigate my website

d) Throughout my html pages I do not use a single in-line styling tag in order to create consistent styling throughout my site. The website itself is modular and thrghout my experience of creating it I have had to modify the pages countless times, proving that it is indeed modular and therefor maintainable.
